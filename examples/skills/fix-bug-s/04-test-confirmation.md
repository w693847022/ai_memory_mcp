## Stage 5: Test Confirmation

**目标**: 确认修复后的测试用例

**流程**:

1. 使用技能`rememory-std` 查询测试规范

2. 展示测试规范摘要：
```
[规范的summary] [规范的content（太长省略用...隐藏后续内容）]
[规范的summary] [规范的content（太长省略用...隐藏后续内容）]
...
```

3. 根据测试规范和确认的修复方案，设计针对本次Bug修复的测试用例

4. 阐述清楚测试步骤、测试用例和边界情况：
   - **Bug复现测试**: 验证Bug能在原条件下复现
   - **修复验证测试**: 验证修复后Bug不再出现
   - **回归测试**: 确保修复不会引入新的问题

5. 要求用户确认
   - 如果用户要求修改则根据用户修改后回到步骤3重新确认测试内容
