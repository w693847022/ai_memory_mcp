## 阶段 4: 方案设计

**目标**: 探索代码库并设计方案

**流程**:
1. 探索：
   - 使用 Explore 代理探索
   - 分析现有架构和模式
   - 设计 1-3 个实现方案
2. 向用户展示方案选项

**方案格式**:
```
## 方案一: <方案名称>

**优点**: <优点列表>
**缺点**: <缺点列表>
**实现步骤**:
1. <步骤1>
2. <步骤2>
...

**影响范围**: <文件列表>

---

## 方案二: <方案名称>
...
```

**🔍 自动检测**:
```
检测内容: 已设计 1-3 个实现方案
检测方式: 检查方案数量和每个方案的必要字段
检测通过标准:
  ✓ 方案数量 >= 1 且 <= 3
  ✓ 每个方案包含: 名称、优点、缺点、实现步骤、影响范围
```

**检测失败处理**: ⏸️ **检测失败，暂停等待用户处理**
- 失败原因: 方案数量不足或方案内容不完整
- 处理方式: 请用户补充方案或完善方案内容（需要包含名称、优点、缺点、实现步骤、影响范围）
