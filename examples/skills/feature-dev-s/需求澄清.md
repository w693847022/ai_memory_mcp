## 阶段 2: 需求澄清

**目标**: 基于相关性探索结果，通过多轮提问，明确功能需求细节

**流程**:
1. 分析用户提供的功能描述
2. 结合相关性探索结果，识别：
   - 已明确的事项（跳过）
   - 仍需澄清的模糊点
   - 边界条件、依赖关系
3. 每轮提问 2-4 个**针对性**问题
4. 用户回答后,进行信息整合后思考模糊点，如果有回到3歩

**提问原则**:
- **优先利用历史信息**: 如果相似记录中已有明确信息，不再重复询问
- **聚焦差异点**: 针对当前需求与已有记录的差异进行提问
- **渐进式深入**: 先问核心问题，再问细节

**澄清完成标准**:
- 功能边界清晰
- 输入输出明确
- 技术约束确认

**🔍 自动检测**:
```
检测内容: 用户已回答所有澄清问题，需求边界清晰
检测方式: 检查澄清完成标准是否全部满足
检测通过标准:
  ✓ 功能边界已明确描述
  ✓ 输入/输出格式已确认
  ✓ 技术约束已确认
  ✓ 用户确认需求已清晰
```

**检测失败处理**: ⏸️ **检测失败，暂停等待用户处理**
- 失败原因: 需求澄清未完成，功能边界、输入输出或技术约束仍有缺失
- 处理方式: 请用户继续回答澄清问题，直到需求清晰
