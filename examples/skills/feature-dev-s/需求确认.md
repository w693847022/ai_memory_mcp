## 阶段 3: 需求确认与记录

**目标**: 确认需求

### 3.1 确认需求

输出需求摘要供用户确认：
```
## 需求确认

**功能名称**: <名称>
**功能描述**: <一句话描述>

**详细需求**:
- <需求点1>
- <需求点2>
- ...

```

**暂停** 并请用户确认以上需求是否正确？

**🔍 自动检测**:
```
检测内容:用户确认需求并且Feature 记录已创建或复用
检测方式: 调用 project_get(project_id, group_name="features", item_id="<feature_id>") 验证
检测通过标准:
  ✓ feature_id 已获取
  ✓ project_get 返回有效记录
  ✓ 记录包含 summary、content、status 字段
  ✓ status 为 "in_progress"
```

**检测失败处理**: ⏸️ **检测失败，暂停等待用户处理**
- 失败原因: Feature 记录创建失败或 feature_id 无效
- 处理方式: 请用户确认 project_add 操作是否成功，或检查 project_id 是否正确

**输出进度**:
```
✅ 阶段3检测通过

实现进度：
[✓] 回忆项目规范
[✓] 项目确认
[✓] 相关性探索
[✓] 需求澄清
[✓] 需求确认 [检测通过]
[→] 方案设计
```
