namd: git feature merge (功能开发分支合并主干)
description: 用于项目功能开发完成后合并到主干分支 
context: fork

---

# 原则:

    - 需要在git操作前确保无冲突文件.
    - 如果需要解决冲突文件需要进入计划模式,列出解决方案.

--- 

# 流程前确认

    - 检测当前分支没有冲突文件,有则列出冲突文件,并退出流程.
    - 需要向用户确认需要合并到主干的开发分支名称
        - 列出当前所有feature开头的分支
        - 询问用户需要合并的开发分支是那个
    - 需要向用户确认主干分支名称
        - 默认为 main

---

# 合并流程

    1. 切换到主干分支,切换失败找出原因后退出流程.
    2. 在主干分支下拉取远程分支确保本地最新,然后检测有无冲突文件,冲突则列出冲突文件后退出流程.
    3. 向用户确认合并分支.
    4. 合并分支到主干.
    5. 检测有无冲突,有冲突则列出冲突文件
        - 进入计划模式,给出解决冲突的方案
        - 用户确认后解决冲突
        - 确认冲突解决
    6. 推送合并后的主干分支到远程仓库.
    7. 询问用户是否需要标记开发分支合并然后删除开发分支.
        - 是:
            - 则添加git tag 格式为:archive/开发分支名称
            - 将tag 推送到远程
            - 删除本地开发分支
        - 否:
            - 退出流程
