## Stage 2: Bug 探索

**目标**: 探索Bug的根因和影响范围

**流程**:

1. **Bug信息收集**
   - 分析用户提供的Bug描述
   - 确认Bug出现的时间、频率、触发条件
   - 如果有错误日志，收集并分析

2. **代码范围探索**
   - 回忆是否有类似Bug的修复记录
   - 探索相关代码文件，理解代码逻辑
   - 确认Bug所在的文件、函数、代码行
   - 追踪数据流和调用链

3. **根因分析**
   - 分析代码逻辑找出Bug产生的根本原因
   - 区分是新功能引入的Bug还是已有功能损坏

4. **影响范围评估**
   - 确认Bug影响的功能范围
   - 确认是否有其他模块受影响
   - 评估修复的紧急程度

---

## 展示格式

```
Bug描述: <用户提供的Bug描述>

触发条件: <触发Bug的条件>

根因分析: <根本原因>

影响范围:
  - 文件: <受影响文件>
  - 函数: <受影响函数>
  - 调用链: <调用链路>

严重程度: <critical/high/medium/low>

修复紧急度: <评估>
```
